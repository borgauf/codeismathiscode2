Basic fractions on the computer

What we call fractions in everyday life and are written in the form \(\frac{a}{b}\) are either meant to represent a ratio, i.e., \(a : b\) and spoken \(a\) to \(b\) or \(a\) is to \(b\), or, more commonly, to imply \(a \div b\), i.e., \(b\) divides (into) \(a\). Although we normally call the outcome of division \(a \div b\) the quotient (with a possible remainder), a fraction as a whole is called a quotient as well

\begin{align*} \text{quotient}\;\left\{ \quad \begin{array}{l} \underline{1} & \leftarrow \text{dividend or numerator} \\[-5pt] 2 & \leftarrow \text{divisor or denominator} \end{array} \right. \end{align*}

Rational numbers are numbers that are expressed in the fraction form \(\frac{a}{b}\) where both \(a\) and \(b\) are integers and \(b \ne 0\). That means, e.g., \(\frac{\sqrt{2}}{2}\) is not a rational number, but is still considered a fraction. And so we can call any rational number a fraction, but not all fractions are rational numbers.1 A fraction can be resolved into a decimal fraction, e.g., \(\frac{3}{4}\) is \(0.75\). But even repeating decimal, e.g., \(\frac{1}{3} = 0.333\ldots\) where the number to the right of the decimal is an infinite series, i.e., \(\frac{3}{10} + \frac{3}{100} + \frac{3}{1000} \ldots\). The set of rational numbers is denoted by \(\mathbb{Q}\), which, again, stands for quotient.

Let’s start with something presumably simple like how to do fractions—specifically, adding, subtracting unlike denominator fractions—on the computer. We’ll first dive into the world of number theory and its concentration on the behavior of prime numbers. So what is the working definition of a prime whole number?

‚•§ A prime number is an integer that can only be divided evenly (producing no remainder) by \(1\) or itself.

The opposite of a prime number is a composite number, i.e., an integer that is a multiple of other numbers beside \(1\) and the number, let’s call it \(n\), itself. With composites there does exist a number, call it \(d\), between \(1\) and \(n\) that will evenly divide it. There are many consequences of these two number opposites. One is the fact that integers are made up of primes

Every positive integer \(ùëõ \gt 1\) can be represented in exactly one way as a product of prime powers.

But we can also “get” any \(n \in \mathbb{N}\) by multiplying sets of primes. Hence, we can almost say, prime numbers necessitate (bring into existence?) multiplication.

‚á≤ Definition Divides

If \(a, b \in \mathbb{Z}\), the integers, we say that \(a\) divides \(b\), written \(a \mid b\), if \(a \cdot c = b\) for some \(c ‚àà \mathbb{Z}\). In this case, we say \(a\) is a divisor of \(b\). We say that \(a\) does not divide \(b\), written \(a \nmid b\), if there is no \(c ‚àà \mathbb{Z}\) such that \(a \cdot c = b\).

Division is the inverse of multiplication, e.g., \(\frac{6}{3} = x\) is identical to \(x \cdot 3 = 6\) as we’ve seen in basic algebra. Defining divides in terms of multiplication by another whole number implies division without a remainder. \(2 \mid 6\) (\(2\) divides \(6\)) and \(-3 \mid 15\) (\(-3\) divides \(6\)). All integers divide \(0\), but only \(0\) divides \(0\). This is why division by \(0\) is problematic. \(q = \frac{a}{0}\) would imply \(q \cdot 0 = a\), but if \(a\) is not zero, then this is nonsense.

‚á≤ Definition Identity

An identity is a type of equality relating one mathematical expression \(A\) to another mathematical expression \(B\), such that \(A\) and \(B\) (which might contain some variables) produce the same value for all values of the variables within a certain domain of discourse.

Unfortunately, two symbols are used to indicate identity. For example

\begin{align*} (a + b)^2 = a^2 + 2ab + b^2 \end{align*}

uses the \(=\) equals sign to indicate identical values are guaranteed given any specific \(a\) or \(b\).5 In Haskell, everything has a value. This is an abstraction meant to keep things highly mathematical, i.e., above imperative programming where evaluating code leads to state changes. With Haskell, the evaluation of code yields values intentionally not involved with state changes. Another identity would be

\begin{align*} sin^{2}\theta + cos^{2}\theta = 1 \end{align*}

Again, we mean the left- and right-hand sides are identical mathematically, as in, both sides have identical values, although they’re obviously different symbols.6 The proof/derivation of \(sin^{2}\theta + cos^{2}\theta = 1\) is usually included in trigonometry texts. This is just one proven identity. But the general idea of term rewriting, i.e., the art of redoing an expression into an identical one, is an important field in math and computability. Which means they are interchangeable. Which means when solving a trig problem, you can substitute in one for the other. Keep track of the subtle difference between is defined as and identical values.

Having said all this, the Haskell operator == and its opposite /= are actually functions that take two formal arguments and return a “decision” of whether they are or are not identical. So what does this Boolean mean? A Boolean is a type of truth value. While the set of natural numbers \(\mathbb{N} = \{0,1,2,\ldots\}\) is endless, the Boolean set is made up of just two elements, \(\{\top, \bot\}\) (logic symbols for top or truth and bottom or falsity) or in Haskell simply True and False.

In the type signature for divides we see the two input arguments n and d, but why does the code ...rem n d == 0 return as its value either a True or False. Because in the realm of Haskell, everything has a value. While \(sin^{2}\theta + cos^{2}\theta = 1\) is indeed an identity, what is the value of this identity? Here we enter the realm of mathematical logic. While the value of the left-hand side, \(sin^{2}\theta + cos^{2}\theta\), becomes whatever \(\theta\) we plug in, and the value of the right-hand side, \(1\), is just \(1\), the value of the identity as a whole is either true or false. A least divisor

‚á≤ We define \(LD(n)\) as the least natural number greater than \(1\) that divides \(n\), or the least divisor of \(n\).

Every number has at least one divisor greater than unit \(1\).10 Again, a prime number has exactly one divisor greater than \(1\), i.e., the prime number itself. Therefore, every number \(n\) has a smallest, least divisor, \(LD(n)\), in its set of divisors, even if that set has only one element. We will leverage this fact to test a number for primeness. Read on.

Let’s look at a composite number \(12\) with the set of divisors \(\{2,3,4,6,12\}\). Here, \(LD(12) = 2\) and the cardinality11 The cardinality of a set is the count of the unique elements contained in the set. of the set is \(5\), i.e., greater than \(1\). What about \(120\)? The divisor set12 …or the factors of \(120\). would be much larger

\begin{align*} \{2,3,4,5,6,8,10,12,15,20,24,30,40,60,120\} \end{align*}

and we can get it with this Haskell code13 This is the Haskell version of set comprehension. More later.

As with \(12\), we see a pattern where we start with a divisor that is prime, then (if any) composite numbers follow.14 Let’s not forget the Fundamental Theorem of Arithmetic, i.e., that \(120\) can also be expressed as product of powers of those first three primes. In both examples notice that the square the \(LD(n)\), e.g., with \(120\) squaring \(2\) is \(4\), is less than \(120\). Why is this germane?

Consider the composite number \(143\). Its divisors are two prime numbers, \(11\) and \(13\). Now, \(11^2 = 121\), while \(13^2 = 169\), i.e., the square of the former, i.e., the \(LD(143)\), is less than \(143\). Neat observation, but does this help us in testing a number for primeness? Yes, and we’ll find out why below.

‚á≤ Proposition

  1. If \(n > 1\) then \(LD(n)\) is a prime number.
  2. If \(n > 1\) and \(n\) is not a prime number, then \((LD(n))^2 \le n\)

The gist of these two implications will be turned into code that will help test if a number is prime.

Why concentrate on the least divisor? The first part of our proposition will anchor the fact that the least divisor is always a prime. This will be necessary for when a number is analyzed for primeness and no divisor less than the number itself has been found. But we need to prove that least divisor is indeed prime. How? We’ll employ a proof method called contradiction15 See proof by contradiction. where we claim just the opposite of what we want to prove, then expose it as not true. In this case, let’s assume \(c = LD(n)\) and \(c\) is not prime, a contradiction of our original statement. But if \(c\) is not prime it must then be a composite number itself, e.g., \(c = a \cdot b\) where \(1 \lt a \lt c\). But that would mean \(a\) now divides \(n\), and \(a\) is smaller than \(c\) the supposed \(LD(n)\). Therefore, a \(LD(n)\) must be prime.

Now, before attacking the second part, let’s do an example.

ùñüùï≠: Is \(599\) a prime?

If \(599\) is composite, then \(599 = d \cdot e\) with \(d,e \gt 1\). This implies

\begin{align*} d &\le \sqrt{599} \quad or \\ e &\le \sqrt{599} \end{align*}

because if both \(d\) and \(e\) were greater than \(\sqrt{599}\) (the opposite of less than or equal to) that would make \(d \cdot e\) greater than \(\sqrt{599} \cdot \sqrt{599} = 599\), which it can’t be. So if \(599\) is composite, it has a divisor between \(2\) and \(\lfloor \sqrt{599} \rfloor\)16 The floor (\(\lfloor\; \rfloor\)) of \(\sqrt{599}\)? First, get the square root of \(599\), which is \(24.474476501040833\); so the lowest whole number, the floor, would be \(24\). The \(\lceil\sqrt{599}\rceil\), or the ceiling of the square root of \(599\) would be \(25\). or \(24\).

At this point we can guess that to show \(599\) is prime, we need to show it has no divisor not a composite number. \(24^2 = 576\), the floor of \(\sqrt{599}\), but \(25^2 = 625\) which is overshoot. We can restate the second part of the proposition thusly

‚á≤ Proposition
Let \(n\) be a natural number. If \(n\) is composite then \(n\) has a prime divisor between \(2\) and \(\lfloor\sqrt{n}\rfloor\). Thus, if \(n \gt 1\) and \(n\) has no prime divisors between \(2\) and \(\lfloor\sqrt{n}\rfloor\), then \(n\) must be prime.

which allows us to quit looking for a \(LD(n)\), i.e., some prime, long before we get to \(599\). If we test this algorithmic proposition on \(599\) we see that none of the primes between \(2\) and \(24\), namely the set of primes \(\{2,3,5,7,11,13,17,19,23\}\) divides \(599\) evenly; therefore, \(599\) is prime.

We now have our main tool for establishing primality, namely, \(LD(n)\) proven. Let’s put it into code. We’ll break it into two parts 

ld :: Integral t => t -> t
ld n = ldf 2 n
ldf k n | divides k n = k
        | k^2 > n = n
        | otherwise = ldf (k+1) n

ld is just a wrapper to feed our number in question into ldf, the “least divisor function” which will do the actual work. What we see with ldf is Haskell’s equation guarding, which allows us to set up test cases. ldf first tries divides (which yields either True or False) on the incoming number argument n using the argument k. We see ldf begins with the lowest prime \(2\) for k. So if \(2 \mid n\) then we’re through, the \(LD(n) = k\). But if it is not then we move on to the next case, namely, Is k squared greater than n?17 This is the second part of \(LD(n)\) Proposition, i.e., we’ve flipped the \((LD(n))^2 \le n\). ld looks for a prime divisor of \(n\) by checking \(k \mid n\) for all \(k\) with \(2 \le k \le \sqrt{n}\). But isn’t a \(LD(n)\) supposed to be prime? We will improve on ld so it doesn’t waste time checking non-prime \(k\) within \(2 \le k \le \sqrt{n}\). Again, this yields a True or False. If True, then n is the least divisor. But if this second test case also fails, we go onto the third, which is not a normal test case, but a hardwired True, i.e., we’re going with this case—because there are no other possible situations. What’s unique about this last case is that it simply calls itself, but with k incremented by \(1\). In other words, we have a repeating mechanism whereby we can march up through the whole numbers one after the other testing each. This is recursion, and we’ll soon be exploring it in depth. But for now just accept that ldf is calling itself, driving inside itself, so to speak, to try another time.

reduce ::  (Integral a) => a -> a -> Ratio a
{-# SPECIALISE reduce :: Integer -> Integer -> Rational #-}
reduce _ 0              =  ratioZeroDenominatorError
reduce x y              =  (x `quot` d) :% (y `quot` d)
                           where d = gcd x y

numerator   (x :% _)    =  x
denominator (_ :% y)    =  y

gcd             :: (Integral a) => a -> a -> a
gcd x y         =  gcd' (abs x) (abs y)
                   where gcd' a 0  =  a
                         gcd' a b  =  gcd' b (a `rem` b)
