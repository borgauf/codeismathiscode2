Logic, Gentzen, and the Natural Numbers

\begin{align*} \hline zero : \mathbb{N} \end{align*} \begin{align*} \text{m} : \mathbb{N} \\ \hline \text{suc m} : \mathbb{N} \end{align*}

This is Inference Rule notation. This style of writing rules (horizontal line with hypotheses above and conclusions below) was popularized by Gerhard Gentzen in his systems of Natural Deduction and Sequent Calculus (1934).

2. The Base Case: zero : ℕ

--------
zero : ℕ
  • Your Interpretation: “A conclusion with no hypothesis.”
  • Correct! In logic, this is called an Axiom.
  • It means: “We don’t need any prior evidence to conclude that zero is a natural number. It is true by definition.”

3. The Inductive Step: suc m : ℕ

m : ℕ
---------
suc m : ℕ
  • Your Interpretation: “An implies / conditional.”
  • Correct! This is a rule of inference.
  • It says: “If you can provide a proof (an element) that m is a ℕ, then I can conclude that suc m is also a ℕ.”

4. The Agda Constructor: suc : ℕ → ℕ

In the Agda code, we write: 

data ℕ : Set where
  zero : ℕ
  suc  : ℕ → ℕ

Can → be understood as logical implication?

Absolutely. In Agda (and Type Theory), the function arrow A → B is identical to the logical implication \(A \implies B\).

  • As a Program: suc is a function that takes a number and returns a new number.
  • As a Logic: suc is a proof that “If ℕ exists, then ℕ exists.”

5. Why this matters: The Curry-Howard Isomorphism

This is the “Magic” of Agda. Every part of the data declaration has a dual meaning:

Agda Term Programming View Logical View
Set A Type A Proposition (Statement)
zero A Value A Proof of the Axiom
suc A Function An Implication / Inference
m : ℕ Variable m has type ℕ m is a proof that ℕ is true
suc m Applying a function Modus Ponens (using the rule)

6. Summary

  • Gentzen’s Rules are the “blueprints” for the logic.
  • Agda’s Constructors are the “implementation” of those rules.

When you write suc (suc zero), you are simultaneously:

  1. Constructing the number 2.
  2. Writing a formal proof that the number 2 exists, based on Gentzen’s rules.

In Agda, Type Checking is Proof Checking. If your code compiles, it means your logical argument is sound.

How suc behaves

suc is not built-in. It is just a constructor that we define when we declare ℕ: 

data ℕ : Set where
  zero : ℕ
  suc  : ℕ → ℕ

Agda’s standard library also defines ℕ with zero and suc, which is why it can feel “built-in”, but logically it’s just an ordinary datatype.

1. What does suc : ℕ → ℕ mean (programming view)?

It is a function:

  • input: a natural number
  • output: a natural number

Example: 

one : ℕ
one = suc zero

two : ℕ
two = suc (suc zero)

2. What does ℕ → ℕ mean (logic view)?

Under Curry–Howard:

  • a type is a proposition
  • a value is a proof

So:

  • ℕ is a proposition
  • ℕ → ℕ is the proposition “ℕ implies ℕ”

BUT here is the subtle (and important) correction:

👉 suc is not a proof that “ℕ exists”.

Instead:

👉 suc is a rule for building proofs of ℕ from other proofs of ℕ.

3. Reinterpreting the inference rule

Your Gentzen-style rule: 

m : ℕ
---------
suc m : ℕ

means:

“If you already have a proof m of ℕ, then suc m is another proof of ℕ.”

So:

  • m is evidence that ℕ holds
  • suc transforms that evidence into new evidence

4. How → corresponds to implication

In logic:

  • implication introduction says: “If assuming A lets you derive B, then A → B holds.”

In Agda:

  • a function f : A → B is exactly that: it takes a proof of A and produces a proof of B

So: 

suc : ℕ → ℕ

means:

Given a proof of ℕ (some number m), we can construct another proof of ℕ (namely suc m).

5. Concrete analogy

Think of ℕ as “the set of all valid constructions of natural numbers.”

  • zero: a base proof (axiom)
  • suc: a rule that extends any existing proof

So: 

zero            -- proof of ℕ
suc zero        -- another proof of ℕ
suc (suc zero)  -- another proof of ℕ

Each number is literally a derivation tree:

  • leaves: zero
  • steps: applications of suc

6. Important clarification

Your earlier phrasing: > “If ℕ exists, then ℕ exists”

is a bit misleading.

Better phrasing:

  • suc witnesses that ℕ is closed under successor
  • i.e.: \[\forall m \in ℕ,\; suc(m) \in ℕ\]

And that is exactly what ℕ → ℕ expresses.

7. Big picture

  • Constructors = inference rules
  • Values = proofs
  • Functions = implications

So suc is:

  • a function (programming)
  • an inference rule (logic)
  • a proof of closure (mathematics)

All at the same time.

How Agda “Builds” Numbers

In a language like C or Python, the number 1 is a bit pattern in memory. In Agda, the number 1 is a symbolic tree.

1. Constructors are “Tags,” not Calculations

When you define: 

data ℕ : Set where
  zero : ℕ
  suc  : ℕ → ℕ

You are telling Agda: “A natural number is either the symbol zero OR it is the symbol suc wrapped around another natural number.”

  • zero is a constant.
  • suc is a “wrapper.”

2. How suc creates recursively

The recursion is in the Type Definition. Look at the type of suc: it takes a ℕ as input. Where does that ℕ come from? It must come from one of the constructors of ℕ.

This creates a “nesting” effect:

  • Level 0: zero
  • Level 1: suc (zero)
  • Level 2: suc (suc (zero))
  • Level 3: suc (suc (suc (zero)))

Agda doesn’t “do” anything to these. It just holds them as a chain of applications. The “value” of the number is simply how many times suc is wrapped around zero.

3. Why must you type suc zero for 1?

You know this because of the Type Signature of the constructors.

If you want to create a value of type ℕ, you look at your “menu” (the constructors):

  1. You could pick zero. (Result: zero : ℕ)
  2. You could pick suc. But suc is a function (ℕ → ℕ). To make it a complete value of type ℕ, you must give it an argument of type ℕ.

The “Substitution” Logic:

  • I want a ℕ.
  • I choose suc.
  • Now I need a ℕ to fill the hole: suc ( ? ).
  • I look at the menu again. I choose zero.
  • Result: suc zero.

This is the only way to satisfy the types. If you tried to type suc 5, Agda would complain because 5 isn’t defined yet! You have to build everything from the ground up using only zero and suc.

4. The “Built-in” Shortcut (Pragmas)

Typing suc (suc (suc (suc zero))) for the number 4 is exhausting. Because this definition is so standard, Agda provides a “shortcut.”

If you tell Agda: 

{-# BUILTIN NATURAL ℕ #-}

Agda will let you type the literal 4, and it will automatically translate it into suc (suc (suc (suc zero))) behind the scenes.

5. Summary: The “Lego” Analogy

  • zero is a base plate.
  • suc is a brick that can snap onto a base plate OR onto another brick.

How do you build a tower of height 2?

  1. Take the base plate (zero).
  2. Snap a brick on it (suc zero).
  3. Snap another brick on that (suc (suc zero)).

The “number” is just the height of the tower you built.

Where is the “recursion” really coming from?

The definition 

data ℕ : Set where
  zero : ℕ
  suc  : ℕ → ℕ

does NOT describe a process. It describes a set closed under rules.

1. Logical reading (rules = closure conditions)

Your inference rules: 

--------
zero : ℕ

m : ℕ
---------
suc m : ℕ

mean:

  • zero is in ℕ
  • whenever m is in ℕ, then suc m is also in ℕ

This is exactly the definition of ℕ in logic:

\[ ℕ = \text{the smallest set such that:} \]

  • \(zero \in ℕ\)
  • if \(m \in ℕ\) then \(suc(m) \in ℕ\)

That phrase “smallest set closed under these rules” is the missing piece.

2. Where recursion comes from

The recursion is hidden in:

> “if m ∈ ℕ then suc(m) ∈ ℕ”

because:

  • to get new elements, you must already have elements
  • the only starting point is zero

So the only possible elements are:

  • zero
  • suc zero
  • suc (suc zero)
  • …

There is no other way to build elements.

This is not an algorithm—it’s a closure property + minimality.

3. Why this matches Haskell’s `Succ Nat`

Haskell: 

data Nat = Zero | Succ Nat

Agda: 

zero : ℕ
suc  : ℕ → ℕ

These are actually the same idea:

  • `Zero` ≈ `zero`
  • `Succ Nat` ≈ `suc : ℕ → ℕ`

The “consing” intuition is:

  • `Succ` / `suc` = a constructor that extends an existing value

So:

  • `Succ (Succ Zero)` = `suc (suc zero)`

4. The deeper logic: inductive definitions

This is called an inductive definition.

Formally:

  • ℕ is the initial algebra generated by:
    • one constant (zero)
    • one unary operation (suc)

Or in proof theory terms:

  • these rules generate the least set closed under them

5. Why you must build numbers this way

Because Agda enforces:

> The only inhabitants of ℕ are those built from its constructors.

So if you want something of type ℕ:

  • you must use either `zero` or `suc`
  • and if you use `suc`, you must supply another ℕ
  • and that ℕ must itself be built the same way

This forces a finite construction:

  • eventually you must stop at `zero`

That’s your “recursion”—but it’s really well-founded construction.

6. The “consing” analogy (precise version)

Think of:

  • List:
    • `[]`
    • xs`
  • Nat:
    • `zero`
    • `suc n`

So:

  • `::` adds an element to a list
  • `suc` adds one “layer” to a number

You can literally see ℕ as a list of unit elements:

\[ 2 \equiv suc(suc(zero)) \sim [(), ()] \]

7. The real punchline

There is no “process” because:

👉 The definition is not operational (not about computation)

👉 It is declarative: “These are exactly the objects that exist.”

And from that, the recursive structure falls out automatically.

8. One-sentence intuition

  • Logic view: “ℕ is the smallest set closed under zero and suc”
  • Programming view: “ℕ is everything you can build using zero and suc”
  • Your intuition: “suc is like cons”

All three are the same statement.

Smallest set closed under these rules…

What “closed under these rules” means (the closure property)

A set \(S\) is closed under the rules

  • \(zero \in S\)
  • \(m ∈ S \implies suc(m) \in S\)

if and only if, whenever you start with \(zero\) and keep applying \(`suc\) any finite number of times, the result is still in S.

Examples of closed sets:

  • The set \(\{0, 1, 2, 3, \dots\}\) (the usual naturals)
  • The set \(\{0, 1, 2, 3, \ldots \} \cup \{extra\, junk\, like\, \frac{1}{2}\, or\, \infty \}\)
  • The set of all terms built from \(zero\) and \(suc\) (even if we imagine a bigger universe)
  • Even the set of everything (the whole universe of sets) is closed.

So there are many closed sets. Closure just says: “don’t leave any generated numbers out”.

What “smallest” or “minimality” means

…is among all possible closed sets, there is one that is contained in every other closed set. That one is called the smallest (or the least fixed point, or the intersection of all closed sets).

Formally:

\[ \mathbb{N}\ :=\ \bigcap\ \{S\ \mid\ S\text{ is closed under zero and suc}\} \]

Why is this intersection itself closed? Because if you take zero (it’s in every \(S\), so in the intersection) and if \(m\) is in the intersection (so \(m\) is in every closed \(S\)), then \(suc(m)\) must also be in every closed \(S\), therefore \(suc(m)\) is in the intersection.

Why is it the smallest? By construction: any other closed set \(T\) contains the intersection.

How this is exactly what Agda’s data declaration encodes

When you write in Agda: 

data ℕ : Set where
  zero : ℕ
  suc  : ℕ → ℕ

Agda is not just giving you two functions. It is declaring:

  • The only way to build an element of ℕ is by starting with zero and applying suc a finite number of times. This is the closure part: Every term you can actually construct satisfies the rules.
  • There are no extra elements. This is the minimality part. Agda guarantees that ℕ is the least such type: any other type that has a zero and a suc map must contain a unique way to map from your ℕ into it.

In type-theory speak: ℕ is the initial algebra for the signature (one constant together with one unary operation). That is precisely the categorical translation of “smallest set closed under the rules”.

### 4. Why this gives you induction “for free”

Because `ℕ` is the smallest closed set, any property P that

  • holds for zero, and
  • is preserved by suc (i.e. if P(n) then P(suc n))

must hold for every element of `ℕ`. If it didn’t, the set {n : ℕ | P(n)} would be a strictly smaller closed set, contradicting minimality.

That is exactly why Agda can give you the built-in induction principle:

```agda ℕ-ind : ∀ (P : ℕ → Set) → P zero → (∀ n → P n → P (suc n)) → ∀ n → P n ```

### 5. Concrete illustration (what would violate minimality?)

Suppose someone tried to define a bigger type:

```agda data Badℕ : Set where zero : Badℕ suc : Badℕ → Badℕ extra : Badℕ – ← this is the “junk” ```

This is closed, but it is not the smallest. Agda refuses to let you do this with `data` (it would be a different type). The real `ℕ` has no `extra`.

### Summary in one sentence

“Smallest set closed under the rules” = the intersection of all sets that contain zero and are closed under successor. Agda’s `data ℕ` is that intersection, packaged as a type with exactly the two constructors. That is why the type is inductive, why pattern matching covers everything, and why induction is valid.

So your source is 100 % correct: it is not an algorithm that generates numbers step-by-step; it is a declarative definition via closure + minimality. Agda turns that mathematical definition into a first-class type you can compute with and prove about.
